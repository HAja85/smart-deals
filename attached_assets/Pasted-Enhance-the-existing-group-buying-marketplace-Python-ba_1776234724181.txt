Enhance the existing group-buying marketplace (Python backend) with:

1) Payment Gateway Integration
2) Refund Handling
3) Reports & Analytics
4) Email & Mobile OTP Authentication
5) Notifications for upcoming deals

System is for Kuwait. Use KWD currency.

=====================
PAYMENT INTEGRATION
=====================

Integrate a payment gateway (Stripe preferred for sandbox, or abstract service layer).

Create PaymentService with methods:

- create_payment_intent(order_id, amount)
- authorize_payment(order_id)
- capture_payment(order_id)
- cancel_payment(order_id)
- refund_payment(order_id)

Flow:

1. When user places order:
   - Create order with payment_status = "Pending"
   - Call create_payment_intent

2. On payment success:
   - Mark payment_status = "Authorized"

3. When deal becomes Successful:
   - Capture all authorized payments

4. When deal Failed:
   - Cancel or refund all payments

Store:
- transaction_id
- payment_method
- payment_status
- paid_at

IMPORTANT:
- Use sandbox mode
- Handle payment failure gracefully
- Ensure idempotency (avoid double charge)

REFUND SYSTEM

Refund scenarios:

1. Deal Failed → full refund

Add fields:

- refund_amount
- refund_time

Create API:
- POST /refund/{order_id}

Logic:
- Only allow refund if payment_status = Captured
- Call payment gateway refund API
- Update refund_status

OTP AUTHENTICATION
=====================

Add Email + Mobile OTP verification during signup/login.

Fields:
- email
- phone_number
- is_verified

Create OTP system:

OTP Model:
- user_id
- otp_code
- expiry_time (5 minutes)
- is_used

APIs:

1. POST /send-otp
- send OTP to email and phone

2. POST /verify-otp
- validate OTP
- mark user as verified

Implementation:

- Email: use SMTP (Gmail or SendGrid)
- SMS: use Twilio (or mock service for MVP)

Rules:
- OTP expires in 5 minutes
- Max 3 attempts
- Resend allowed after 60 seconds
