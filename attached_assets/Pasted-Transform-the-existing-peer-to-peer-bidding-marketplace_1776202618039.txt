Transform the existing peer-to-peer bidding marketplace into a modern group-buying marketplace for supermarket deals in Kuwait.

=====================
CORE ARCHITECTURE
=====================

Keep Product as master entity. Introduce Deal as child entity.

1. Product (existing - keep as is)
- id
- title
- category
- description
- image
- brand (NEW - for supermarket use)
- unit (e.g., carton, pack, kg)
- seller_id

2. Deal (new model)
- id
- product_id (FK)
- seller_id
- target_quantity
- current_quantity (default 0)
- price_per_unit
- start_time
- end_time
- status (Active, Successful, Failed)

3. Order (replace bidding)
- id
- user_id
- deal_id
- quantity
- total_amount
- payment_status (Pending, Authorized, Captured, Cancelled)
- created_at

=====================
USER ROLES
=====================

Seed and enforce 2 roles:

1. Supplier (Seller)
- Can create products
- Can create deals

2. Consumer (Buyer)
- Cannot create products
- Cannot create deals
- Can only:
    - browse products/deals
    - place orders

Enforce role-based access in APIs.

=====================
BUSINESS LOGIC
=====================

1. Join Deal
- Consumer selects quantity
- Validate:
    - deal is Active
    - current time < end_time
- Create order
- Increase:
    deal.current_quantity += quantity

2. Deal Success
- If current_quantity >= target_quantity BEFORE end_time:
    - status = Successful
    - update all orders → payment_status = Captured

3. Deal Failure
- If end_time reached AND target not met:
    - status = Failed
    - update all orders → payment_status = Cancelled

=====================
BACKGROUND JOB
=====================

Create scheduled job (runs every minute):

For each Active deal:
    IF now >= end_time:
        IF current_quantity >= target_quantity:
            mark Successful
        ELSE:
            mark Failed
